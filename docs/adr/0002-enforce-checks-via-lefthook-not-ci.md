# Enforce checks with a lefthook pre-commit hook, not CI

The repo has three quality gates (`make fmt-check`, `make lint`, `make test`) but nothing ran them automatically — they were opt-in, which is fine for a careful human and useless as a guardrail against agent-authored changes. We now run all three as a **pre-commit hook managed by lefthook** (`brew "lefthook"`, a `lefthook.yml`, installed by `script/setup` via `lefthook install`), and we **deliberately do not add CI**.

The workflow is direct commits to `master` (no PR flow). CI-on-push therefore runs *after* a change has already landed on `master` — it notifies, it cannot prevent — whereas a pre-commit hook gates the change at the exact point an agent or human is about to record it. That makes local enforcement strictly more useful here and CI redundant-to-ceremony, so the earlier PRDs' "CI: out of scope" stays true while "no enforcement at all" does not. lefthook (over a raw `.git/hooks` script or the Python `pre-commit` framework) was chosen for the declarative, managed, husky-like DX the maintainer knows from JS, installed through Homebrew to match how the repo already adopted StyLua — no npm, no Python runtime. lint-staged-style staged-file filtering was considered unnecessary: the full suite runs in under two seconds on a six-file repo.
