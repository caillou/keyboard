# Docs and LICENSE

## PRD

[2026-05-29-lua-dev-tooling](../prd.md)

## What to build

Document the new toolchain and update the legal/authorship notices, now that the
tools and editor config exist to describe accurately.

- **CLAUDE.md — reframe "Deliberate non-choices"** from a list of prohibitions
  into a single case-by-case principle: dependencies are evaluated on their
  merits, balancing not-invented-here against needless complexity, favoring good
  developer experience and current best practices. Drop the purely dogmatic
  entries (e.g. the `ReloadConfiguration` Spoon note).
- **CLAUDE.md — keep the one informative non-choice**: why `windows.lua` is
  hand-rolled rather than a stock window-manager Spoon (custom layouts like
  `left40`, `right60`, `centerWithFullHeight` those Spoons cannot do, so swapping
  one in is a regression). Frame it as recorded rationale, not dogma. Under the
  new principle the vendored EmmyLua.spoon needs no carve-out — note it as a
  dev-only tooling Spoon for editor autocomplete.
- **CLAUDE.md — add an "Editor and tooling" section** documenting StyLua, LuaLS,
  luacheck, the no-conflict tool separation, and the EmmyLua autocomplete stubs:
  how they regenerate and how to refresh the generator (`make update-emmylua`).
- **README — add a short tooling pointer**: format with `make fmt`, lint with
  `make lint`, and that opening the repo in VS Code prompts for the recommended
  extensions and gives `hs.*` autocomplete. Keep it brief.
- **LICENSE — add the maintainer's copyright line** alongside the original
  author's, under the existing MIT terms, retaining the original notice:

  ```
  Copyright (c) 2013 Jason Rudolph (http://jasonrudolph.com)
  Copyright (c) 2022-2026 Pierre Spring <pierre.spring@caillou.ch>
  ```

This layers on top of the space-fn PRD's own CLAUDE.md/README edits rather than
duplicating them.
