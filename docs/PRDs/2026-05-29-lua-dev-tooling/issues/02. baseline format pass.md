# One-time baseline format pass

## PRD

[2026-05-29-lua-dev-tooling](../prd.md)

## What to build

Run StyLua once across the whole repo and commit the result as a clean baseline,
so future diffs are never polluted by an untouched file getting reformatted the
first time someone opens it. Because `stylua.toml` matches the existing style,
this should be a near-no-op.

Show the diff to the maintainer before committing so they can confirm it is the
expected near-no-op. (This is the human-in-the-loop step — the diff is reviewed,
not merged blind.)

## User stories covered

- 1 — Lua files auto-formatted (the baseline brings the whole tree to conformance).
- 3 — one-time baseline format committed up front.
- 4 — the baseline-format diff is shown before it is committed.

## Acceptance criteria

- [ ] StyLua has been run across the entire repo.
- [ ] The resulting diff is presented to the maintainer for confirmation before any commit.
- [ ] The reviewed baseline is committed as a single clean baseline.
- [ ] `make fmt-check` passes on the resulting tree.
