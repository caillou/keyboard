## PRD

[2026-05-29-lua-dev-tooling](../prd.md)

## What to build

The final review found that CLAUDE.md's `.luacheckrc` description (around line
52, the "luacheck — the linter" bullet in the "Editor and tooling" section) is
now incomplete after the lint-clean follow-up work. It documents that formatting
checks (`max_line_length`, the W6xx family) are off and that `hs` is declared a
read global, but it no longer fully reflects the current `.luacheckrc`, which
additionally has:

1. An `exclude_files` entry excluding `hammerspoon/Spoons/` and `lua_modules/`
   (mirroring `.styluaignore`), so luacheck never lints vendored/installed code.
2. A declared block of intentional module-level globals (`configWatcher`,
   `keyboardWatcher`, `keyUpDown`, `enableHotkeyForWindowsMatchingFilter`).
3. A per-file override making `hs.window` writable in `hammerspoon/windows.lua`
   (the documented layout-functions-attached-to-`hs.window` pattern).

Update the CLAUDE.md bullet so the description matches the shipped
`.luacheckrc`. This is LOW priority, docs-only, no behavior change. Note that
the `.luacheckrc` itself is already well-commented, so this is purely keeping
the prose doc in sync.
