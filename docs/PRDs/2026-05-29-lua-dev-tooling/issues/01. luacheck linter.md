# luacheck linter: install, config, make target

## PRD

[2026-05-29-lua-dev-tooling](../prd.md)

## What to build

Add luacheck as the logic-only linter (the eslint analog), installed the same
way busted is and configured so it never contradicts StyLua. luacheck owns
unused variables, undefined globals, shadowing, and unreachable code — and
nothing about formatting or whitespace.

- Install luacheck into the project-local tree alongside busted via
  `luarocks --tree=lua_modules install luacheck`, added to `script/setup`
  next to the busted install line. Idempotent.
- Commit a `.luacheckrc` at the repo root that:
  - declares `hs` as a read global so `hs.*` references are not flagged undefined;
  - applies busted's standard set to `spec/` so `describe`, `it`, `assert` and
    friends are recognized there;
  - disables formatting and whitespace diagnostics, including `max_line_length = false`,
    so StyLua owns line length and spacing (the luacheck analog of `eslint-config-prettier`).
- Add a `make lint` target that runs luacheck from `lua_modules/`.

## User stories covered

- 11 — `make lint` reports only logic issues (unused vars, undefined globals, shadowing, unreachable code).
