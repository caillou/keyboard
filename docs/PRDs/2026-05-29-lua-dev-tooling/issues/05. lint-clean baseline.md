# Lint-clean baseline (resolve remaining luacheck warnings)

## PRD

[2026-05-29-lua-dev-tooling](../prd.md)

## What to build

- `make lint` is currently permanently red: luacheck reports ~51 logic/style
  WARNINGS (0 errors) in existing source, so the linter gives little signal — a
  real future bug would hide in the expected noise. There is no lint-clean
  baseline the way StyLua got one in the baseline-format pass. Bring `make lint`
  to a clean (exit 0) baseline.
- Resolve the warnings with a mix of the two correct tools, NOT by disabling
  logic checks wholesale:
  - (a) Declare genuinely-INTENTIONAL globals/patterns in `.luacheckrc` (with
    explanatory comments), e.g. the deliberate globals (`configWatcher`,
    `keyboardWatcher`, `keyUpDown`, `enableHotkeyForWindowsMatchingFilter`) and
    the documented pattern of attaching layout functions onto `hs.window` (the
    "setting read-only field of global hs" warnings in windows.lua) — these are
    intentional per CLAUDE.md and should be allowed, not flagged.
  - (b) FIX the genuine issues in source: missing `local` (e.g.
    `currentScreenIndex`, `nextScreenIndex`, `windowLayoutMode`,
    `getModifiersStr` used as implicit globals in windows.lua), unused variables
    (e.g. `log` in init.lua, `geometry` in status-message.lua, unused loop vars
    `i`), variable shadowing (status-message.lua `messageText`/`seconds`;
    windows.lua `modifiers`/`trigger`), and empty/dead branches (e.g. space-fn.lua
    empty if-branch). Preserve behavior exactly.
- This deliberately goes slightly beyond the original PRD's "no source
  refactoring beyond baseline format" scope — it is a maintainer-approved
  follow-up.

## Acceptance criteria

- [ ] `make lint` exits 0 (no warnings, no errors) on the whole tree.
- [ ] Intentional globals/patterns are allowed via `.luacheckrc` (commented), not by turning off logic checks globally.
- [ ] Genuine issues (missing local, unused vars, shadowing, empty branches) are fixed in source without behavior change.
- [ ] busted still passes 15/0; `make fmt-check` still clean; `make fmt` still idempotent.
