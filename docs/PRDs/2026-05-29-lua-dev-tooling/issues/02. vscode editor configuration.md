# VS Code committed editor configuration

## PRD

[2026-05-29-lua-dev-tooling](../prd.md)

## What to build

Commit the VS Code configuration so opening the repo prompts for the right
extensions and applies the no-conflict settings automatically. This is what
turns the installed tools into the on-save / autocomplete experience: StyLua
formats, LuaLS provides IntelliSense and diagnostics with its own formatter off,
and the EmmyLua stubs feed `hs.*` autocomplete.

- **`.vscode/extensions.json`** recommends `sumneko.lua` and
  `JohnnyMorganz.stylua`. VS Code prompts to install on open; clicking "Install"
  is the one unavoidable manual step (it cannot silently auto-install).
- **`.vscode/settings.json`** sets at minimum:
  - StyLua as the default formatter for Lua, with format-on-save enabled for Lua.
  - `Lua.format.enable = false` so the language server does not also format.
  - `hs` declared as a known global, and busted's globals recognized for `spec/`.
  - A `package.path`-equivalent so `require('keyboard.…')` resolves from `spec/`
    (mirroring the `.busted` `./hammerspoon/?.lua` entry), so go-to-definition
    works spec-to-source.
  - `Lua.workspace.library` pointing at the EmmyLua annotations via the
    workspace-relative path `${workspaceFolder}/hammerspoon/Spoons/EmmyLua.spoon/annotations`
    — portable (no hard-coded username) and symlink-free on the read side,
    provided the editor is opened at the repo's real location.

## User stories covered

- 7 — editor autocomplete for the `hs.*` API.
- 8 — hover documentation and signatures for `hs.*` calls.
- 9 — go-to-definition and diagnostics from the language server.
- 10 — language server's own formatter disabled.
- 19 — prompted to install recommended extensions on open.
- 20 — committed settings apply automatically (StyLua formatter, format-on-save, LuaLS formatter off, hs/busted globals).
- 21 — autocomplete library path is portable (no hard-coded username, no broken symlink resolution).

## Acceptance criteria

- [ ] `.vscode/extensions.json` recommends `sumneko.lua` and `JohnnyMorganz.stylua`; opening the repo prompts to install them.
- [ ] StyLua is the default Lua formatter with format-on-save enabled; saving a Lua file formats it via StyLua and LuaLS does not re-format or fight the result.
- [ ] `Lua.format.enable` is `false`.
- [ ] `hs` and busted globals are known to LuaLS (no false undefined-global diagnostics, including in `spec/`).
- [ ] A spec-to-source `require('keyboard.…')` resolves for go-to-definition.
- [ ] `Lua.workspace.library` uses the workspace-relative annotations path; typing `hs.window.` offers autocomplete with signatures.
