# luacheck + VS Code config cleanup

## PRD

[2026-05-29-lua-dev-tooling](../prd.md)

## What to build

- The vendored EmmyLua Spoon is excluded from StyLua via `.styluaignore` but
  NOT from luacheck, so `make lint` now also lints
  `hammerspoon/Spoons/EmmyLua.spoon/init.lua` (vendored upstream code we don't
  own). Add an `exclude_files` entry to `.luacheckrc` that excludes
  `hammerspoon/Spoons/` (and `lua_modules/` if not already implicitly excluded),
  mirroring `.styluaignore`, so luacheck never lints vendored/installed code.
  Use luacheck's `exclude_files = { ... }` mechanism (globs).
- `.vscode/settings.json` has a redundant `${workspaceFolder}/hammerspoon` entry
  in `Lua.workspace.library`. Module/spec resolution is already handled by
  `Lua.runtime.path`, and putting in-workspace source into `workspace.library`
  is the wrong mechanism (treats editable source as a read-only library). Remove
  that one entry. KEEP the EmmyLua annotations library entry
  (`${workspaceFolder}/hammerspoon/Spoons/EmmyLua.spoon/annotations`) — that one
  is correct and required.
