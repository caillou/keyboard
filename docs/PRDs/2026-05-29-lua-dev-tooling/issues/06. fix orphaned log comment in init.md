# Fix orphaned log debug comment in init.lua

## PRD

[2026-05-29-lua-dev-tooling](../prd.md)

## What to build

Commit `0fb1c50` removed the unused `local log = hs.logger.new(...)` from
`hammerspoon/init.lua` but left a debug-instruction comment in `keyUpDown`:

```lua
-- Un-comment & reload config to log each keystroke that we're triggering
-- log.d('Sending keystroke:', hs.inspect(modifiers), key)
```

Following that instruction (uncommenting the `log.d(...)` line) references an
undefined `log` and throws a nil-index error. Remove the dead instruction +
`log.d(...)` comment lines so no path a user can follow from the comments leads
to an undefined-`log` error. A live keystroke logger already exists in
`space-fn.lua` for anyone who actually needs one, so the hook here adds no value.
