# Add .editorconfig, version pinning, and make ergonomics

## PRD

[2026-05-31-dev-tooling-hardening](../prd.md)

## What to build

Round out the non-Lua surfaces and developer ergonomics that the existing toolchain misses, holding to the repo's minimal-ceremony principle.

- **`.editorconfig`** at the repo root sets, at minimum, 2-space indentation, UTF-8, final newline, and trailing-whitespace trim for the file types StyLua does not own (markdown, yaml, toml). It must not fight StyLua over `.lua` files — StyLua stays the sole Lua formatter.
- **Pin rock versions.** `script/setup`'s LuaRocks install lines pin busted and luacheck to exact versions, so a fresh clone installs the same tool versions and a diagnostic-changing luacheck upgrade cannot silently break the pre-commit hook. No `.rockspec` is introduced; the pin lives in the install command.
- **Make ergonomics.** Add `reload` (`hs -c "hs.reload()"`), `console` (open the Hammerspoon console), and a coverage target running busted with luacov.
- **Gitignore luacov output** (e.g. `luacov.stats.out`, `luacov.report.out`) so machine-specific coverage reports are never committed.

The luacov target measures the *existing* Engine spec — it confirms which Engine state transitions the spec exercises, supporting ADR-0001's push-logic-into-the-Engine practice. It adds no new tests.

## User stories covered

- 20: `.editorconfig` gives non-VS-Code editors an indent/charset/newline baseline.
