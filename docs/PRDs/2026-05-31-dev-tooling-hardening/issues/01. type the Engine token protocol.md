# Type the Engine token protocol

## PRD

[2026-05-31-dev-tooling-hardening](../prd.md)

## What to build

Make the Engine's stringly-typed contract a checked, autocompleted type in the editor by adding EmmyLua annotations to `space-fn-engine.lua` only. The Adapter and `windows.lua` are deliberately not annotated — the Engine is where the type *is* the design (the Event-token / Action-token protocol) and where the contract is stable and spec-pinned, so drift risk is lowest and payoff highest.

Declare:
- The Event tokens (`space-down`, `space-down-autorepeat`, `space-up`, `key-down`, `key-up`, `timer-fire`) as a `---@enum`.
- The Action tokens (`emit-space`, `emit-key`, `emit-remap`, `suppress`, `passthrough`, `start-timer`, `cancel-timer`) as a `---@enum`.
- `---@class` shapes for the action objects that carry payload fields (e.g. `emit-key`'s `key`/`mods`, `emit-remap`'s `remap`/`physMods`) and for a keymap entry.
- `---@param`/`---@return` on `M.new(opts)` and `M:advance(event, payload)`.

This is comments plus an editor setting — **no Engine behavior changes**. Whether the Engine merely documents the tokens or also references the enum tables in its branch conditions is an implementation detail, provided the existing spec passes unchanged and no token is rejected at runtime (runtime token validation is explicitly out of scope). Set LuaLS diagnostics in `.vscode/settings.json` to a moderate level — enough to surface undefined fields and bad token strings, but not the strictest setting, which floods false positives against the `setmetatable({}, { __index = M })` object pattern. The precise diagnostic keys are an implementation detail; the intent is "useful, low-noise."

## User stories covered

- 11: mistyped Event token (e.g. `'space-dwon'`) is underlined, not silently defaulted.
- 12: Action tokens are a checked enum, so Adapter and spec share a typo-proof protocol.
- 13: action objects (`emit-key`, `emit-remap`) have declared payload shapes.
- 14: the keymap-entry shape is declared.
- 15: `new()` and `advance()` are annotated with param/return types.
- 16: LuaLS diagnostics set to a moderate, low-noise level.
- 17: annotations change no Engine behavior; the spec passes unchanged.

## Acceptance criteria

- [ ] Event tokens and Action tokens are each declared as a `---@enum` in `space-fn-engine.lua`.
- [ ] Payload-carrying action objects and the keymap entry have `---@class` shapes.
- [ ] `M.new` and `M:advance` carry `---@param`/`---@return` annotations.
- [ ] `make test` passes unchanged — no Engine behavior altered, no token rejected at runtime.
- [ ] In the editor, a mistyped Event/Action token string and an undefined action field are underlined.
- [ ] The `setmetatable` object pattern does not produce a wave of false positives at the chosen diagnostic level.
- [ ] Only `space-fn-engine.lua` is annotated; the Adapter and `windows.lua` are untouched.
