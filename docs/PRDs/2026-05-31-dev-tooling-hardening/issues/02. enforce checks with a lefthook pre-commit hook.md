# Enforce checks with a lefthook pre-commit hook

## PRD

[2026-05-31-dev-tooling-hardening](../prd.md)

## What to build

Make the existing quality checks *guaranteed* rather than opt-in by gating them at the commit boundary — the exact point an agent or human is about to record a change on the direct-to-`master` workflow. A `lefthook`-managed pre-commit hook runs the full check suite before a commit is recorded.

- A root **`lefthook.yml`** defines a `pre-commit` stage running, as parallel commands, the checks that gate a commit: `fmt-check` (StyLua check), `lint` (luacheck + shellcheck), and `test` (busted). Every check that gates a commit must be represented; the exact grouping (whether shellcheck is its own command or carried inside `lint`) is left to implementation.
- The **whole suite runs**, not a staged-file subset — the suite is sub-two-seconds on this repo, so lint-staged-style filtering is omitted as needless complexity.
- **`brew "lefthook"`** is added to the Brewfile (Homebrew flow — no npm, no Python runtime).
- **`script/setup` runs `lefthook install`** idempotently after `brew bundle`, so the hook is wired on a fresh clone with no manual git-hook step. Re-running setup stays safe.
- **`git commit --no-verify`** must remain available as an intentional bypass.
- **CI is explicitly not added** (ADR 0002): a push-triggered run reports after a change has already landed on `master`, so it notifies but cannot prevent. The commit boundary is the gate.

lefthook is chosen over a raw `.git/hooks` script or the Python `pre-commit` framework for the declarative, managed, husky-like DX (ADR 0002).

## User stories covered

- 1: checks have already run before an agent-authored change is committed.
- 2: a forgotten `local` (accidental global) blocks the commit.
- 3: a broken Engine state transition fails `make test` inside the hook.
- 4: StyLua format drift blocks the commit.
- 5: the hook is installed automatically by `script/setup` on a fresh clone.
- 6: the hook is managed by a declarative `lefthook.yml`, not a hand-rolled script.
- 7: lefthook is installed through `brew bundle`.
- 8: the hook runs the whole suite, not a staged-file subset.
- 9: `git commit --no-verify` bypass remains available.
- 10: CI is deliberately not added.
- 19: shellcheck runs inside the pre-commit hook.

## Acceptance criteria

- [ ] A root `lefthook.yml` defines a `pre-commit` stage running fmt-check, lint (luacheck + shellcheck), and test as parallel commands.
- [ ] `brew "lefthook"` is in the Brewfile and installs via `brew bundle`.
- [ ] `script/setup` runs `lefthook install` idempotently after `brew bundle`; re-running setup is safe.
- [ ] A staged commit with StyLua drift, a luacheck finding, a failing Engine spec, or a shellcheck finding is blocked by the hook.
- [ ] `git commit --no-verify` bypasses the hook.
- [ ] The hook runs the whole suite (no staged-file filtering).
- [ ] No CI / GitHub Actions workflow is added.
