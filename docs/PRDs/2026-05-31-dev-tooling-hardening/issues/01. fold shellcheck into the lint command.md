# Fold shellcheck into the lint command

## PRD

[2026-05-31-dev-tooling-hardening](../prd.md)

## What to build

Bring the install scripts under static analysis. shellcheck runs over `script/setup` and `script/update-emmylua` (and any future `script/*`), and is folded into `make lint` so the single lint command covers both Lua and shell — matching exactly what the pre-commit hook will later run.

shellcheck is installed through the existing Homebrew flow (a `brew` entry), introducing no new dependency mechanism. Its defaults are accepted as-is: no project config file is added. Any genuinely-needed suppression is an inline `# shellcheck disable=` directive at the offending line, never a blanket config. The scripts must end up clean under shellcheck's default checks.

## User stories covered

- 18: shellcheck guards the fragile install path (cliInstall half-states, PATH repair).
- 26: shellcheck folded into `make lint` so one command checks Lua and shell.

## Acceptance criteria

- [ ] shellcheck is added to the Brewfile and installs via `brew bundle`.
- [ ] `make lint` runs shellcheck over `script/*` in addition to the existing Lua lint.
- [ ] `make lint` reports both Lua and shell findings and exits non-zero on either.
- [ ] A deliberately broken shell line in a `script/*` file is flagged by `make lint`.
- [ ] `script/setup` and `script/update-emmylua` pass shellcheck with defaults (no config file); any needed suppression is an inline `# shellcheck disable=` directive.
