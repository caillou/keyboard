# Set up test toolchain

## PRD

[2026-05-29-space-fn](../prd.md)

## What to build

Bring in busted (the Lua test framework) via LuaRocks, installed into a project-local `lua_modules/` tree so nothing leaks into global package state. The setup script should pick this up on a fresh clone with no extra steps. A top-level `Makefile` becomes the public entry point for both setup and tests, so contributors (and future-me) don't have to remember which script-vs-make convention applies.

This issue produces no application behavior. Its job is to make the next issue's test suite runnable.

## User stories covered

- 23. Single Makefile entry point for setup and tests.
- 28. `script/setup` installs the test toolchain idempotently on a fresh clone.

## Acceptance criteria

- [ ] `Brewfile` includes `luarocks` and `lua`.
- [ ] `script/setup` runs `luarocks --tree=lua_modules install busted` after `brew bundle`, and is idempotent on re-run.
- [ ] `.busted` at the repo root adds `./hammerspoon/?.lua` to `package.path` so specs can `require('keyboard.space-fn-engine')`.
- [ ] `Makefile` exposes `make install` (delegates to `./script/setup`), `make test` (runs `./lua_modules/bin/busted`), and `make test-watch` (runs `./lua_modules/bin/busted --watch`).
- [ ] `.gitignore` excludes `lua_modules/`.
- [ ] `make install && make test` on a fresh clone succeeds (with a placeholder or empty spec — full suite arrives in the next issue).
